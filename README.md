# FIPI Bank Solve Platform

Это платформа для решения задач из открытого банка заданий ЕГЭ и ОГЭ сайта ФИПИ.

## Основные компоненты

*   **Парсер**: Утилита для сбора заданий по указанному предмету с сайта `ege.fipi.ru`.
*   **API**: Бэкенд на FastAPI для предоставления доступа к задачам и проверки ответов.
*   **База данных**: Хранилище на основе SQLAlchemy для задач и данных пользователей.
*   **Фронтенд**: Vue.js приложение для взаимодействия с пользователем.

## Новые возможности и улучшения

### Аутентификация и отслеживание прогресса пользователя

В приложение добавлена полноценная система аутентификации и авторизации:
*   **Регистрация/Вход/Выход**: Пользователи могут регистрироваться, входить и выходить из системы, используя свой email, никнейм и пароль. Подтверждение email не требуется.
*   **Управление сессиями**: Использование `fastapi-users` с `CookieTransport` и `JWTStrategy` для управления сессиями. Сессии хранятся в HTTP-only cookie, имеют срок жизни в 7 дней и автоматически продлеваются при активном использовании.
*   **Отслеживание решенных задач**: Для зарегистрированных пользователей сохраняется прогресс — список `task_id` всех корректно решенных задач.
*   **Решение без регистрации**: Пользователи могут решать задачи без регистрации, но их прогресс не будет сохраняться.

### Функционал задач и главной страницы

*   **Проверка ответов**: Добавлена возможность ввода ответа для задач с кратким ответом и его проверки через API ФИПИ.
*   **Отметка "Выполнено"**: Для задач, не требующих краткого ответа, добавлена кнопка "Отметить как выполненное", которая сохраняет задачу в прогресс пользователя.
*   **Фильтрация и пагинация**:
    *   **Поиск по коду КЭС**: На главной странице реализован фильтр по кодам КЭС, что позволяет пользователям выбирать задачи из определенных тематических разделов.
    *   **Фильтрация по типу задачи**: Задачи можно фильтровать по типу ("Часть 1" - с кратким ответом, "Часть 2" - остальные).
    *   **Пагинация**: Задачи выводятся постранично (по 15 задач на страницу), реализовано переключение между страницами.
*   **Исключение решенных задач**: Для авторизованных пользователей из основного списка задач автоматически исключаются уже решенные задачи, что помогает сосредоточиться на новых.
*   **Улучшенное отображение изображений**: Исправлено позиционирование изображений рядом с текстом задания, что уменьшает пустое пространство и улучшает читаемость. Добавлен белый фон для прозрачных GIF-изображений.
*   **Русская локализация**: Весь фронтенд переведен на русский язык.

### Панель управления (Дашборд)

Полностью переработан дашборд пользователя:
*   **Вкладки "Статистика" и "Решенные задачи"**: Удобное переключение между общими метриками и детальным списком решенных зада.
*   **Статистика**: Отображает общее количество решенных задач и статистику по каждому коду КЭС.
*   **Решенные задачи**: Выводит список всех задач, решенных пользователем, с возможностью просмотра деталей задачи.

## Начало работы

1.  **Установите зависимости**:
    ```bash
    # Убедитесь, что у вас установлен uv
    # pip install uv
    uv pip sync app/pyproject.toml
    npm install --prefix frontend
    ```
2.  **Запустите парсер (если база данных пуста)**:
    ```bash
    python app/parser/run_parse.py
    # Возможно, потребуется загрузить задачи из JSON в БД:
    # python app/utils/load_json_to_db.py 
    ```
3.  **Запустите бэкенд**:
    ```bash
    uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
    ```
    API будет доступно по адресу `http://127.0.0.1:8000`. Интерактивная документация (Swagger UI) доступна по адресу `http://127.0.0.1:8000/docs`.

4.  **Запустите фронтенд**:
    ```bash
    npm run dev --prefix frontend
    ```
    Фронтенд будет доступен по адресу `http://localhost:5173`.

## Будущие улучшения

### Оптимизация потребления памяти при парсинге

**Проблема:** Текущая реализация парсера (`app/parser/src/parser.py`) загружает все задания с сайта ФИПИ в оперативную память и только после этого сохраняет их в один JSON-файл. При большом количестве заданий это может привести к критическому потреблению ОЗУ и зависанию системы.

**Решение:** Необходимо провести рефакторинг процесса парсинга для использования **потоковой записи в JSON**. Это означает, что:
1.  Выходной JSON-файл открывается в самом начале.
2.  В него записывается открывающая скобка `[`, обозначающая начало массива.
3.  Каждая спарсенная задача **сразу же** записывается в файл (с добавлением запятой-разделителя), после чего освобождается из памяти.
4.  После обработки всех задач в файл дописывается закрывающая скобка `]`.

Такой подход гарантирует, что в памяти одновременно будет находиться лишь небольшая порция данных, что обеспечит низкое и стабильное потребление ресурсов.
