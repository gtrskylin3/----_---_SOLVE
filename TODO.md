Идея: создать удобную платформу для решения задач егэ по профильной математике с сайта фипи для этого нужно

## Этап 1: Парсинг заданий

### 1.1. Анализ существующего парсера
- Изучить код парсера с GitHub, чтобы понять его логику и возможности.
- Определить, можно ли его адаптировать для наших целей или лучше написать свой.

### 1.2. Разработка собственного парсера
- **Навигация**:
    - Отправная точка: `https://ege.fipi.ru/bank/`.
    - Переход к заданиям по математике (профильный уровень) по ссылке `https://ege.fipi.ru/bank/index.php?proj=AC437B34557F88EA4115D2F374B0A07B`.
    - Задания находятся в `iframe` с `src` `https://ege.fipi.ru/bank/questions.php?proj=AC437B34557F88EA4115D2F374B0A07B&init_filter_themes=1`.
- **Извлечение данных**:
    - Для каждого задания необходимо извлечь:
        - Текст вопроса.
        - Уникальный GUID вопроса (из `id` элемента или `onclick` атрибута).
        - Номер задания.
        - Категорию (КЭС).
        - Тип ответа ("Краткий ответ" или "Развернутый ответ").
        - Ссылки на изображения, если они есть.
- **Сохранение данных**:
    - Сохранить собранные данные в `json` файл. Каждый объект в `json` должен представлять одно задание со всеми извлеченными атрибутами.

## Этап 2: Проектирование базы данных

### 2.1. Модели SQLAlchemy
- Создать модель `Task` для хранения информации о заданиях. Поля:
    - `id` (primary key)
    - `task_guid` (уникальный GUID с FIPI)
    - `task_number` (номер задания в КИМ)
    - `category` (КЭС)
    - `task_type` ("short" или "detailed")
    - `task_text` (текст задания)
    - `image_url` (ссылка на изображение, если есть)
- Создать модель `User` для хранения информации о пользователях.

## Этап 3: Наполнение базы данных и проверка ответов

### 3.1. Импорт данных из JSON
- Написать скрипт для парсинга `json` файла и сохранения данных в базу данных `sqlite` с использованием моделей SQLAlchemy.

### 3.2. Механизм проверки ответов
- Для заданий с кратким ответом реализовать функцию, которая отправляет `POST` запрос на `https://ege.fipi.ru/bank/solve.php`.
- **Параметры запроса** (`multipart/form-data`):
    - `guid`: GUID задания.
    - `answer`: Ответ пользователя.
    - `ajax`: `1`.
    - `proj`: `AC437B34557F88EA4115D2F374B0A07B`.
- **Обработка ответа**:
    - Ответ сервера: `3` - правильно, `2` - неправильно.

## Этап 4: Разработка API

### 4.1. FastAPI эндпоинты
- Создать эндпоинт для получения списка задач с возможностью фильтрации по номеру в КИМ и категории КЭС.
- Создать эндпоинт для отправки ответа на проверку. Он будет принимать `task_guid` и `answer`, вызывать функцию проверки и возвращать результат.

## Стек технологий
- **Python**: Основной язык разработки.
- **BeautifulSoup4 (bs4)**: Для парсинга HTML.
- **requests**: Для отправки HTTP запросов.
- **FastAPI**: Для создания API.
- **SQLAlchemy**: Для работы с базой данных.
- **Pydantic**: Для валидации данных.
- **uv**: Для управления зависимостями и виртуальным окружением.
- **sqlite**: В качестве базы данных.
- **Chrome DevTools (MCP)**: Для анализа сайта.
